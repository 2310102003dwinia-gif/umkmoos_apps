
# UI Guidelines for **UMKMoo's** Project

## 1. **General Design Principles**
### 1.1 Consistency
- Maintain consistent styling across all screens and components.
- Use a unified color palette, font styles, and button shapes.
- Keep the layout consistent in terms of spacing, margins, and padding.

### 1.2 Simplicity
- Focus on simplicity by keeping the interface clean and user-friendly.
- Use minimal text and icons to convey the message clearly.
- Avoid unnecessary animations or complex interactions that could confuse the user.

### 1.3 Accessibility
- Ensure that all text has sufficient contrast for readability.
- Make sure interactive elements are large enough for touch interaction.
- Provide support for screen readers and offer alt text for images.

---

## 2. **Color Palette**
### 2.1 Primary Colors
- **Dark Green**: #006400 (used for buttons, links, and primary elements)
- **Light Green**: #A8D08D (used for backgrounds and highlights)
- **White**: #FFFFFF (for background, text, and containers)
  
### 2.2 Secondary Colors
- **Gray**: #A9A9A9 (for secondary text and disabled elements)
- **Light Gray**: #F5F5F5 (for backgrounds and borders)
- **Red**: #D9534F (for error messages and warnings)

---

## 3. **Typography**
### 3.1 Font Family
- **Primary Font**: Arial, sans-serif (for headings and body text)
- **Secondary Font**: Helvetica, sans-serif (for buttons and small text)

### 3.2 Font Sizes
- **Headings**: 24px for H1, 20px for H2, 16px for H3
- **Body Text**: 14px for paragraphs
- **Buttons**: 16px for button text

---

## 4. **Layout and Spacing**
### 4.1 Margins and Padding
- Use 16px for outer margins and 8px for inner padding for most components.
- Maintain consistent spacing between components (e.g., buttons, form fields).

### 4.2 Grid System
- Use a 12-column grid for responsive layout on larger screens.
- For mobile devices, switch to a 6-column grid for better content alignment.

---

## 5. **Navigation**
### 5.1 Sidebar Navigation
- **Layout**: Fixed on the left side of the screen.
- **Icons**: Use clear, easy-to-understand icons for each section.
- **Color**: Highlight the active menu item with **Dark Green** color.

### 5.2 Top Navigation Bar
- **Layout**: Fixed at the top with the business name and status (Online/Offline).
- **Menu Options**: Include the "Logout" button at the top right.
- **Search Bar**: Place the search bar in the center for easy access.

---

## 6. **Buttons and Actions**
### 6.1 Button Styles
- **Primary Buttons**: Background color **Dark Green** with white text. Rounded corners (8px).
- **Secondary Buttons**: Background color **Light Gray** with Dark Green text. Rounded corners (8px).
- **Disabled Buttons**: Background color **Gray** with light gray text.

### 6.2 Action Buttons
- **Position**: Place action buttons like "Add" or "Save" at the bottom-right of screens.
- **Icon**: Use a small plus icon for adding, check icon for confirming actions.

---

## 7. **Forms and Inputs**
### 7.1 Input Fields
- **Style**: Rounded corners with a light gray background.
- **Focus**: Highlight the input field with a **Light Green** border on focus.
- **Error State**: Display an error message in **Red** below the field with a clear description.

### 7.2 Drop-down Menus
- Use a clear and concise label for drop-down menus.
- Show an arrow icon indicating the direction of the drop-down.
  
---

## 8. **Icons and Images**
### 8.1 Icon Usage
- **Simple Icons**: Use flat icons that are intuitive and easy to recognize (e.g., shopping cart, user profile, settings).
- **Active State**: Change icon color to **Dark Green** when selected or active.

### 8.2 Image Handling
- Use high-quality images with consistent styling.
- Ensure images have proper alt text for accessibility.

---

## 9. **Responsive Design**
### 9.1 Mobile View
- Ensure all elements are stacked vertically for mobile screens.
- Use collapsible side menus and floating buttons for easier navigation.

### 9.2 Desktop View
- Utilize a multi-column layout for dashboards and reports.
- Allow ample space for graphs, data tables, and navigation.

---

## 10. **Error Handling and Feedback**
### 10.1 Error Messages
- Display error messages clearly in **Red** text.
- Provide a description and suggested actions to resolve the issue.

### 10.2 Success Feedback
- Display success messages in **Light Green** with a checkmark icon to confirm the action was successful.

---

## 11. **Interactive Elements**
### 11.1 Hover Effects
- **Buttons**: Change button background color to **Light Green** on hover.
- **Links**: Underline links and change color to **Dark Green** on hover.

### 11.2 Animations
- **Page Transitions**: Apply smooth fade-in transitions for page changes.
- **Button Animations**: Subtle hover animations (scale effect) for interactive buttons.

---

**End of UI Guidelines for UMKMoo's**
